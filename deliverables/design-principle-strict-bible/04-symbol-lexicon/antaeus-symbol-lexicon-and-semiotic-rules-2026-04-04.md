# Antaeus Symbol Lexicon And Semiotic Rules

This file freezes the visible meaning system behind Antaeus signs, marks, tones, and short-form status language.

It exists to stop the product from inventing decorative signals, fake meters, or inconsistent symbols page by page.

## Governing rule

Every sign must map to a real system truth.

If a mark, color, line, dock, or token suggests:
- urgency
- proof
- health
- staleness
- scale
- ranking

then the underlying product logic must actually support that meaning.

## Primary signal channels

The command rail and dashboard-critical account language now use three fixed channels:

### `Risk`
- Meaning: what is structurally wrong, drifting, overdue, or exposed
- Typical values:
  - `100`
  - `Drift`
  - `Gap`
  - `Log it`
- Tone family: orange-first when the pressure is active and unresolved

### `Proof`
- Meaning: how grounded, qualified, or commercially supported the object is
- Typical values:
  - `Fit`
  - `Thin`
  - `Cover`
  - `$120k`
- Tone family: blue-first because proof is a truth/structure layer, not a threat layer

### `Motion`
- Meaning: whether live work is carrying the object forward right now
- Typical values:
  - `47d`
  - `Owner`
  - `Hot`
  - `Ready`
- Tone family:
  - green when momentum is live and validated
  - muted/cold when motion is stale, missing, or uncarried

## Tone system

### Orange
- Means: pressure, exposed risk, overdue correction, intervention required
- Must not be used as generic excitement

### Blue
- Means: proof, structure, qualification truth, supported framing
- Must not be used as filler accent color

### Green
- Means: live motion, validated heat, healthy movement, real operating momentum
- Must remain rare and earned

### Cold muted blue-gray
- Means: stale, uncarried, latent, not yet activated
- Must not pretend to mean failure by itself

## Mark families

### Risk mark
- Job: indicate exposed pressure or instability
- Rules:
  - must feel alert, directional, or consequential
  - cannot feel playful

### Proof mark
- Job: indicate evidence, structure, qualification, case support
- Rules:
  - should imply layering, framing, or support
  - should not be mistaken for storage/navigation

### Motion mark
- Job: indicate live work, live momentum, or the absence of it
- Rules:
  - when green, it means live carry
  - when cold, it means uncarried or stale motion

## Dashboard command rail rules

The chosen dashboard account language is:
- one account object
- three fixed signal rows
- one right-side dock

This means:
- no chip sprays
- no wrapping token rows
- no fake heat meter
- no secondary sign system competing with the three rows

The right-side dock is not a heat scale.
It is a compact echo of the three fixed rows.

## Token language rules

Short values must be:
- compact
- concrete
- legible in one glance
- stable enough to repeat across accounts

Short values must not be:
- mini headlines
- jargon bursts
- long sentence fragments

## Forbidden semiotic drift

Forbidden:
- a line that looks like a meter but is not tied to a real scale
- a color that changes meaning by room
- symbols that exist only because there is space to fill
- chips that stand in for uncertainty about structure
- decorative accents that imply more system than actually exists

## Current locked dashboard reading

For the canonical dashboard account rail:

- the **left block** is the working object
- the **right dock** is commercial/row-signature support
- the **three rows** are the only first-order signal language

Everything else is subordinate.
