# Antaeus Facial Architecture Rubric

Date: 2026-04-03

Status: canonical judging rubric for all wireframes and visible convergence work

Purpose: define how a proposed Antaeus screen is judged before it earns implementation.

This rubric is for a full visible rebuild, not a cosmetic refresh.

This rubric exists to keep the product from drifting into:

- card soup
- text soup
- hallway behavior
- generic SaaS cleanliness
- architecture-jargon UI
- emotionally flat minimalism

This rubric is not only behavioral.

It also judges:

- beauty
- restraint
- composition
- authority
- product distinctiveness

It assumes the visible design language of the whole product is being rebuilt while the product mind remains intact.

---

## 1. How To Use This Rubric

Every candidate wireframe or prototype is scored across the categories below.

Use:

- `5 = strong pass`
- `4 = acceptable`
- `3 = weak / likely needs revision`
- `2 = failing`
- `1 = hard reject`

Any wireframe that triggers a **hard reject rule** fails even if the average score looks acceptable.

### 1.1 Preservation rule

The governing rule is:

**rewrite the face, not the mind**

So every candidate must improve visible expression without changing:

- module brain
- GTM framework meaning
- strategic logic
- room purpose
- compounding truth

---

## 2. Category A: First-Second Impression

### A1. Distinctiveness

Question:

Does this look like Antaeus, not like a cleaned-up CRM or dashboard bundle?

Pass signals:

- specific visual posture
- authored hierarchy
- obvious identity
- no generic component-library feel

Fail signals:

- looks like a SaaS template
- feels assembled from cards and pills
- could belong to any B2B admin app

### A2. Emotional Territory

Question:

Does the surface feel calm, severe, commercially serious, and alive?

Pass signals:

- high-conviction but not loud
- expensive without ornament
- pressure is legible

Fail signals:

- flat
- over-friendly
- over-gamified
- dark-and-heavy founder admin energy

### A3. Beauty

Question:

Would a sophisticated user feel attracted before being instructed?

Pass signals:

- controlled air
- strong typography
- elegant restraint
- satisfying proportion

Fail signals:

- purely functional but ugly
- no compositional pleasure
- visual weight feels accidental

---

## 3. Category B: Hierarchy And Burden

### B1. Dominant Move Clarity

Question:

Is there one clear next move?

Hard reject:

- more than one visually dominant primary move

### B2. Object Clarity

Question:

Does the surface establish what object or system truth is in focus before controls compete?

### B3. Copy Burden

Question:

Is the first visible zone light enough to scan without becoming shallow?

Pass signals:

- short, consequence-aware copy
- support language subordinate to the action
- deeper explanation deferred

Fail signals:

- essay at the top
- repeating the same point in multiple blocks
- internal product language doing surface work

### B4. Plane Count

Question:

How many dominant visual planes compete in the first visible zone?

Hard reject:

- more than 3 dominant planes

### B5. Box Count

Question:

Is the layout relying on too many bordered containers to create structure?

Hard reject:

- panel soup
- card-after-card hierarchy
- “tile field” composition on a page that should feel authored

---

## 4. Category C: Behavioral Alignment

### C1. Hallway Suppression

Question:

Does the surface steer the user into the work instead of inviting browsing?

Hard reject:

- room/menu browsing competes with the main work

### C2. Continuity

Question:

Do I know where I came from, what I am on, and what this affects next?

### C3. Pressure Legibility

Question:

Can I see why this matters now without reading a memo?

### C4. Progressive Disclosure

Question:

Is deeper methodology moved to the right layer instead of dumped into the top surface?

---

## 5. Category D: Visual System Discipline

### D1. Color Discipline

Question:

Are color roles semantically controlled?

Pass signals:

- bright neutral field
- navy authority
- blue for system intelligence
- orange rationed for pressure and action
- green used sparingly for health/completion

Hard reject:

- decorative chip rainbow
- candy-state greens
- accent overuse that destroys hierarchy

### D2. Typography Discipline

Question:

Do type scales feel authored and limited?

Fail signals:

- too many text sizes
- too many “small all-caps labels” doing nothing
- giant hero text without earned authority

### D3. Container Discipline

Question:

Are borders, fills, and radius used intentionally, not as default scaffolding?

### D4. Surface Economy

Question:

Could one-third of the visible containers disappear while preserving clarity?

If yes, the current wireframe is probably still overbuilt.

---

## 6. Category E: Substance Preservation

### E1. Module-Brain Preservation

Question:

Did the design simplify the visible face without flattening the room’s actual strategic job?

Hard reject:

- the room looks cleaner only because nuance was removed
- the logic was quietly rewritten in the name of simplification

### E2. Strategic Seriousness

Question:

Does the surface still feel like a system behind serious commercial work?

### E3. Methodology Placement

Question:

Is the doctrine still available, but no longer sprayed across the visible working surface?

---

## 7. Automatic Rejection Rules

Reject any wireframe that does any of the following:

- revives flat hallway logic
- uses more than 3 dominant visual planes
- relies on card accumulation as the main ordering system
- uses internal architecture language as visible product copy
- uses green, blue, and orange as decorative candy rather than semantic roles
- makes the product feel more like a generic dashboard than a premium operating instrument
- achieves calm by erasing pressure
- achieves beauty by flattening module substance
- rewrites the mind instead of the face

---

## 8. What A Great Wireframe Should Feel Like

A great Antaeus wireframe should feel like:

- cut down, not dressed up
- severe but not cold
- expensive but not ornamental
- clear before verbal
- strategically alive
- one composed environment

The user should think:

- “I know what this is.”
- “I know why it matters.”
- “I know what happens next.”
- “This feels sharper than the tools I’m used to.”

---

## 9. Review Output Format

For each candidate, record:

1. overall score
2. strongest behavior
3. biggest visual liability
4. whether it passes the beauty bar
5. whether it passes the seriousness bar
6. whether it preserves the room brain
7. keep / revise / reject
