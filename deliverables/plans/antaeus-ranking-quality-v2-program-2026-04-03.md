# Antaeus Ranking Quality v2 Program
Date: 2026-04-03
Status: Active
Owner: Codex / Antaeus restructure continuation

## Purpose
Improve the quality, stability, and trustworthiness of `Brief / Spotlight / Queue` ranking without changing:

- GTM frameworks
- module substance
- methodology
- strategic seriousness
- room brains
- command architecture

This is a ranking-quality wave, not a shell wave and not a methodology rewrite.

## Why now
The restructure program is complete enough that the next leverage point is ranking quality.

Current command intelligence is functional and explainable, but it still has first-pass limitations:

- pressure signals are relatively shallow
- ranking can shift too easily when nearby objects are close
- the surface does not yet distinguish strongly between high-confidence and mixed-confidence winners
- older hallway-era heuristics are gone structurally, but ranking can still feel less believable than the preserved room logic behind it

## Non-goals
This wave does not:

- redesign the shell
- rewrite room logic
- alter Signal Console methodology
- alter Future Autopsy methodology
- alter Discovery / PoC / Advisor strategic flows
- change `Inspect -> Enter room -> Back`
- change the preserved deep-room map

## Program shape

### Phase 1
Pressure + stability scoring

Deliver:

- richer scoring signals
- anti-thrash stability bonus
- confidence labeling for ranked objects
- Dashboard consumption of the new ranked output

### Phase 2
Evidence-aware ranking expansion

Deliver:

- stronger use of proof thinness, champion uncertainty, next-step decay, and room-action readiness
- better distinction between recovery pressure, motion leverage, and truth debt

### Phase 3
Room-fed ranking inputs

Deliver:

- selected room outputs feeding better command quality
- ranking reflects deeper-room truth more directly instead of only surface card summaries

### Phase 4
Acceptance and tuning

Deliver:

- stability review
- ranking sanity checks
- explanation quality review
- no-break continuity review

## Phase 1 success criteria

- `Spotlight` feels more believable
- `Queue` feels less noisy and less jumpy
- a previously valid lead does not get displaced by tiny score fluctuations
- reason tags remain short
- command surfaces remain architecturally unchanged
- Dashboard still preserves return context, sheet flow, graph reward, and pinned continuity

## Definition of done for this program

- ranking quality is materially more trustworthy than v1
- explainability stays compact
- the engine uses deeper pressure signals without bloating UI
- no preserved room brain is altered
- no connective tissue is broken
