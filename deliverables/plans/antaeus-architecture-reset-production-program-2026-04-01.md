## 2. Executive Position

Antaeus is no longer redesigning around prettier modules.

Antaeus is now executing a production architecture reset with this target state:

- command-first entry
- object-based continuity
- one dominant next move
- open loops that migrate forward instead of disappearing
- preserved deep rooms behind a smarter front door
- bright, clean, behaviorally disciplined visual identity

The main thing being replaced is not module substance.
It is the hallway.

The current app already contains strong rooms.
This program changes:

- how the user enters the product
- how pressure is ranked
- how context carries forward
- how modules are accessed
- how the product feels in the first 100 milliseconds

## 4. Non-Negotiable Constraints

### 4.1 No substance dilution

The strategic depth of the modules is preserved.

The reset is not allowed to flatten:

- Discovery Studio frameworks
- Future Autopsy seriousness
- PoC Framework logic
- Advisor Deploy logic
- Signal Console meaning
- Readiness synthesis
- Playbook / Handoff Kit value

### 4.2 No CRM drift

The product must not drift toward:

- record-admin choreography
- generic database tables as the primary experience
- tab forests
- neutralized pipeline software

### 4.3 No panel soup

The product must not solve hallway problems by creating surface noise.

Rejected patterns:

- too many equal-weight cards
- chips everywhere
- nested panels inside panels
- dashboard clutter
- scratch-paper / sticky-note composition

### 4.4 No old-shell visual residue

The old heavy dark-and-warm shell is not the future.

Production implementation must follow the locked bright system:

- brighter base
- cleaner structure
- fewer boxes
- stronger hierarchy
- sharper controls
- calmer field

### 4.5 No simultaneous total collapse

Do not rebuild the entire product in one pass.

The correct path is staged:

- command first
- sheet second
- workspace bridge third
- object migration by family after that

## 17. Visual and UX Enforcement Rules During Production

These rules are mandatory in every production phase.

### 17.1 Plane discipline

No important screen should present more than three dominant visual planes.

### 17.2 Box discipline

If divider lines, spacing, typography, and placement can solve the layout, do not add another card.

### 17.3 Chip discipline

Chips exist to label critical state, not to decorate every surface.

### 17.4 Metric discipline

Each major object surface gets one dominant metric or fact anchor.
That anchor should hit before the explanatory copy.

### 17.5 Text discipline

Top-layer surfaces must not read like brochures.
Detail belongs in sheets and deep rooms.

### 17.6 Action discipline

If a screen has four equal CTAs, it has no CTA.

One primary move.
One fallback move.
Everything else quieter.
