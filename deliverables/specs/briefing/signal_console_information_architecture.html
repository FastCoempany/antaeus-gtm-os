Signal Console · Information Architecture · 17 May 26

Where every surface lives, and how they relate.

Across the seven design phases the product grew from a single Briefing surface into a constellation of related surfaces: Watchlist Triggers, Deal-Watch, Watch List, Sources & Health, plus configuration and integration paths to the rest of the GTM OS. This document maps it.

02 · Always-on

Cross-cutting UI elements.

Persistent on every primary surface. Three of these. They don't change content per surface — they reflect global state.

Top Nav Bar
Primary surface navigation + badges
Five-tab nav: Briefing · Triggers · Deal-Watch · Watch List · Sources. Badges on Triggers (fire count) and Deal-Watch (alert count). Cmd+K opens global search.
Every surface, top
Notification Strip
Today's fires + high-severity alerts
Dismissable strip below the nav. Shows today's trigger fires and high-severity Deal-Watch alerts as a quick-glance summary. Click-through to relevant surface.
Briefing · Triggers · Deal-Watch (collapsible)
Footer Strip
Pipeline health + cost telemetry
Persistent footer. Pipeline last-run timestamp · sources healthy / total · cost this week / ceiling · settings link. Quiet but always visible.
Every surface, bottom
03 · Configuration

Settings cluster.

Accessed from the footer or via Cmd+K. Not primary surfaces — these are where the user configures the system, not where they read intelligence.

SETTINGS · VOICE
Voice Document editor
Edit exemplars, anti-exemplars, banned vocabulary, structural rules, hedging rules. Versioned with diff view.
SETTINGS · CONTEXT
ICP & behavioral profile
Connects to ICP Studio · read-only here. Surfaces "last updated" timestamp + suggestions for refinement.
SETTINGS · DEALS
Active Deals register management
Bulk add / edit / close deals. Same data as Deal-Watch but a management view rather than alert view.
SETTINGS · COST
Cost ceilings & usage
Weekly / daily ceiling configuration. Per-source cost breakdown. Override toggle. Historical cost log.
SETTINGS · FEEDBACK
Behavioral feedback log
Audit of every Used / Met / Noise mark. Reset weights option. Quarterly review prompt.
SETTINGS · TELEMETRY
Pipeline diagnostics
Hydrated-context logs · per-stage timing · enrichment-cost-per-source · evaluation-harness regression flags.
04 · Topology

GTM OS read / write topology.

Signal Console is the intelligence layer. It reads from every relevant GTM OS module at the start of every pipeline run, and writes back via user-confirmed recommended-moves routing. Reads are continuous; writes require explicit approval.

05 · Flows

Four user flows the IA must support.

These are the paths users will walk most often. If any of them feels awkward in the IA above, the IA is wrong.

Monday morning Briefing routine

05
